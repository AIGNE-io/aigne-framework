# 2026 Q1 Roadmap

## Repo 架构重组

### 0. 私有主 Repo + 公开 Artifact 模式

**状态**: 计划中

**背景**:
- 当前在开源 repo 直接开发，被迫使用英文，不符合团队语言习惯
- 开源 repo 应该是发布产物（artifact），不是开发主战场

**目标架构**:
```
arcblock-ai/                    # 私有主 repo (开发主战场)
├── afs/
│   ├── core/                   # public
│   ├── drivers/                # public
│   └── runtime/                # private
├── aigne/
│   ├── framework/              # public
│   ├── runtime/                # private
│   └── observability/          # private
└── aine/                       # private

        ↓ publish (版本发布时)

公开 Repo (artifact, 只读镜像)
├── afs/                        # AFS Core + Drivers
└── aigne-framework/            # AIGNE Framework
```

**执行计划**: [repo-restructure-plan.md](../intents/repo-restructure-plan.md)

**7 天时间线**:
| Day | 任务 |
|-----|------|
| 1 | 创建 repos (afs, aigne, afs-oss, aigne-framework-oss) |
| 2-3 | AFS 拆分：消除对 core 依赖，迁移代码 |
| 4-5 | AIGNE 重组：移除 afs，调整依赖 |
| 6 | 发布脚本，CI/CD 配置 |
| 7 | 团队切换，归档旧 repo |

**团队分工**:
- 每人负责一个 repo，减少冲突
- AI 重构速度快，隔离 chamber 是关键
