AIGNE Framework:
  描述: AIGNE Framework 是一个功能型 AI 应用开发框架，旨在简化和加速现代应用程序的构建过程。它结合了函数式编程特性、强大的人工智能能力和模块化设计原则，帮助开发者轻松创建可扩展的解决方案。AIGNE Framework 还深度集成了 Blocklet 生态系统，为开发者提供丰富的工具和资源。
  核心特性:
    - TypeScript 支持
    - Blocklet 生态系统支持
    - 支持多种工作流模式
  
  框架架构:
    Chat Model:
      - 大语言模型抽象的类类
      - 快速支持多种大语言模型
    
    Agent:
      - 每个 Agent 执行提供特定的能力
      - 指定的输入输出结构
      - 工作流的基础
      子类型:
        AI Agent: 与大语言模型交互的 Agent
        Function Agent: 封装代码逻辑为 Agent
        MCP Agent: 通过 CMP 协议连接外部服务
    
    Workflow:
      - 顺序工作流
      - 独立工作流
      - 交接工作流
      - 反思工作流
      - 代码执行工作流
      - 编辑工作流
    
    Execution Engine:
      - 调度 Agent

  常见问题:
    - 支持哪些大语言模型:
      AIGNE 框架支持扩展，允许您通过实现自定义的 LLMModel 类来与任意 LLM 服务交互
    - 是否有使用示例:
      在我们的 Github 仓库（https://github.com/AIGNE-io/aigne-framework/tree/main/examples）中有多种使用示例
    - 是否有相关的学习资料:
      在我们的官方站点和 Github 仓库（https://github.com/AIGNE-io/aigne-framework/tree/main）中很很多学习资料
    - 如何选择合适的工作流:
      选择工作流模式时应考虑以下因素:
        - 任务复杂度: 简单任务可以使用单一Agent，复杂任务应使用多Agent工作流
        - 交互需求: 需要用户参与的任务适合使用反思或交接工作流
        - 并行性: 可独立执行的子任务适合并发工作流
        - 流程控制: 严格按步骤进行的任务适合顺序工作流
        - 决策分支: 需要根据输入动态选择处理路径的任务适合路由工作流
        - 复杂协调: 需要多个专业Agent协同工作的任务适合编排工作流
    - 工作流可以组合使用吗:
      复杂应用可能需要组合多种工作流模式，工作流可以自由组合
    - 如何集成外部系统和API:
      可以使用MCPAgent连接到相应的MCP服务器或创建自定义FunctionAgent封装API调用
